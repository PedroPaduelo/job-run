🔐 Sistema de Autenticação

Demo Mode — AuthContext.tsx

No modo demo, o AuthContext força automaticamente um token JWT e um organizationId no localStorage ao inicializar a aplicação. Isso permite explorar o painel sem necessidade de cadastro.
// AuthContext.tsx — demo mode init useEffect(() => { const existing = localStorage.getItem( 'nommand_access_token' ) if (!existing) { localStorage.setItem( 'nommand_access_token', DEMO_JWT_TOKEN ) localStorage.setItem( 'organizationId', 'org_demo_001' ) } }, [])

Token Flow — lib/api.ts

// lib/api.ts — interceptor de auth const client = axios.create({ baseURL: import.meta.env.VITE_API_URL, }) client.interceptors.request.use((config) => { const token = localStorage.getItem( 'nommand_access_token' ) if (token) { config.headers.Authorization = `Bearer ${token}` } return config })
chave nommand_access_token — JWT salvo no localStorage
org organizationId — ID da organização para multi-tenancy
guard ProtectedRoute — redireciona /login se sem token

🎨 Tema — Dark Mode

O tema dark é padrão. O ThemeContext adiciona/remove a classe dark no elemento html, ativando as variantes dark: do Tailwind. A preferência é salva em localStorage.
// ThemeContext.tsx const [theme, setTheme] = useState<'dark'|'light'>( () => localStorage.getItem('theme') ?? 'dark' ) useEffect(() => { document.documentElement.classList.toggle( 'dark', theme === 'dark' ) localStorage.setItem('theme', theme) }, [theme])

Configuração Tailwind

// tailwind.config.ts export default { darkMode: 'class', // ← classe no <html> content: ['./src/**/*.{ts,tsx}'], theme: { extend: { // Sem custom colors — usa // as paletas padrão do Tailwind } } } // Uso nos componentes: // bg-white dark:bg-slate-900 // text-slate-800 dark:text-slate-200
