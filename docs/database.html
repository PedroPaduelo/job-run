User

Chave Primária
id String @id
Campos únicos
email String @unique
slug(String) @unique (Organization)
key(String) @unique (ApiKey)
Enumerações
UserStatus: ACTIVE, INACTIVE, SUSPENDED
MemberRole: OWNER, ADMIN, MEMBER, VIEWER

Modelos Principais

🔐 User
• id (PK)
• email (UNIQUE)
• firstName, lastName
• displayName, avatarUrl
• status: enum
• Sincronizado do Auth Service
• relaciona: memberships, jobs, recurringJobs, auditLogs
🏢 Organization
• id (PK de/auth)
• slug (UNIQUE)
• name, logoUrl
• settings (JSON)
• isActive
• relaciona: members, apiKeys, queues, jobs
🔗 OrganizationMember
• userId (FK User)
• organizationId (FK Org)
• role: OWNER, ADMIN, MEMBER, VIEWER
• permissions[]
• Unique(userId, orgId)
🗝️ ApiKey
• id, key (UNIQUE)
• name, description
• permissions[]
• allowedIps, allowedDomains
• rateLimit (rpm)
• isActive, expiresAt
• organizationId (FK)
• usageCount, lastUsedAt
📡 Queue
• id
• name
• concurrency (default 5)
• maxRetries (3)
• retryDelay (ms)
• timeout (ms)
• rateLimit (rpm)
• isActive, isPaused
• Unique(orgId, name)
📦 Job
• id (PK)
• externalId (optional)
• queueId, organizationId
• userId (creator)
• webhookUrl, webhookMethod
• callbackUrl (optional)
• payload (JSON), result (JSON)
• status, priority, attempts
• scheduledAt, startedAt, completedAt
• recurrenceId (FK RecurringJob)
• tags[], metadata (JSON)
📝 JobLog
• Log de cada tentativa
• jobId (FK), attempt, status
• message, duration (ms)
• metadata (JSON)
⏳ RecurringJob
• recurrenceType: INTERVAL | CRON
• interval (minutes) or cronExpression
• webhookUrl, webhookMethod, callbackUrl
• payloadTemplate (JSON)
• maxRuns, recurrenceEndDate
• continueOnFailure
• isActive, nextRunAt
• relaciona jobs[]
📊 AuditLog
• userId, organizationId
• action (string)
• entityType, entityId
• metadata (JSON)
• ipAddress, userAgent
• status: SUCCESS/FAILURE/DENIED

Estratégia Multi-Tenant

TODAS as tabelas (exceto User que sincroniza do Auth) possuem organizationId e todas as queries usam where: { organizationId }. Prisma garante isolamento em nível de aplicação. Não há row-level security do PostgreSQL.

Performance Highlights

  • Grupos por status em filas: prisma.job.groupBy({ by: ['queueId', 'status'] }) evita N+1.
  • Duração média: raw SQL AVG(EXTRACT(EPOCH FROM ...)) em vez de trazer todos registros.
  • Bulk insert: createMany com skipDuplicates para 1000+ jobs.
  • Índices em colunas de filtro e datas.
