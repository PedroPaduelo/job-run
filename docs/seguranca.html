Segurança

🔐 Autenticação e Autorização

JWT (usuários)

  • • Validação: JWKS do Auth Service externo
  • • Cache: JWKS cacheado no Redis por 5min
  • • Claims: sub, email, roles, permissions, org_id
  • • Issuer: Múltiplos aceitos (nommand.com, localhost)

API Keys (M2M)

  • • Criação: Apenas admins/org owner
  • • Permissões: Granular (jobs:create, jobs:read, etc)
  • • Expiração: Opcional (expiresAt)
  • • IP Allowlist: Opcional (allowedIps)

⚠️ Considerações de Segurança

🔓 Demo Mode (Frontend)

O frontend força um token JWT de demo no localStorage. Em produção, isso deve ser removido e o token deve vir do fluxo OAuth real.

🔓 CORS permissivo

Atualmente origin: true — aceita qualquer origem. Em produção, restringir para origens conhecidas.

🔓 Rate limit fallback

Se Redis falhar, rate limiting falha-open (allowed: true). Isso evita indisponibilidade, mas pode permitir ataques DDoS.
