Operação e Deploy

🚀 Como rodar o sistema

Backend (API + Worker)

# Instalar dependências
cd backend-job-assync
npm install
# Rodar migrations
npx prisma migrate deploy
# API Server (pm2)
pm2 start npm --name "job-async-api" -- run start
# Worker (pm2)
pm2 start npm --name "job-async-worker" -- run worker

Frontend

# Instalar + Build
cd frontend-job-async
npm install
npm run build
# Servir (pm2)
pm2 start npx --name "job-async-frontend" -- serve dist/ -l 5173 --no-clipboard
# Dev mode
npm run dev -- --host 0.0.0.0

🆘 Troubleshooting

❌ "Queue is not active" ao criar job

A fila está pausada ou desativada. Acesse /queues no frontend e ative a fila.

❌ "Organization not found"

Token JWT inválido ou organização não existe. Verifique se o organizationId está no token ou header X-Organization-Id.

❌ Jobs ficam em PENDING para sempre

Worker pode estar offline. Verifique pm2 list e logs do worker. Também verifique se Redis está acessível.

❌ Rate limit exceeded

Organização excedeu limite (default 1000/min). Aguarde ou ajuste JWT_RATE_LIMIT_PER_MIN no .env.

❌ Webhook falha sempre

Verifique: URL acessível? HTTPS em produção? Timeout suficiente? Logs do job no frontend mostram o erro.
