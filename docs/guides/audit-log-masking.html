Audit Log Masking

Redação de PII nos logs de auditoria

🎭 O Problema

AuditLog metadata pode conter informações sensíveis (emails, tokens, IPs). É necessário sanitizar antes de salvar para compliance.

✅ Recomendações

  1. Sanitização — filtrar keys sensíveis como token, password, apiKey, email
  2. IP anonimização — salvar /24 em vez de /32
  3. Encrypt metadata — se contém PII, usar pgcrypto ou app-level encryption
  4. Access control — AuditLog já isola por organizationId

🔍 Checklist

  • Metadata sanitizado antes salvar? ❌
  • IPs anonimizados? ❌
  • Encryption em metadata? ❌
  • Access control em AuditLog? ✅
