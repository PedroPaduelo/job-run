🚀 Guia Completo do JobSync

Sistema de processamento de jobs assíncronos com filas, recorrências e monitoramento em tempo real

📖 O que é o JobSync?

O JobSync é um sistema que permite executar tarefas de forma assíncrona e em segundo plano. Em vez de deixar seu usuário esperando uma tarefa demorada terminar (como enviar 10.000 emails, processar um CSV gigante, ou chamar APIs externas), você cria um job e o JobSync processa em filas paralelas, com retries, monitoramento e controle total.

🎯 Quando usar?

  • Envio em massa de emails/notificações — Processa milhares sem travar a aplicação
  • Integrações com APIs externas — ViaCEP, gateways de pagamento, webhooks
  • Processamento de arquivos — CSV, XML, imagens, relatórios
  • Tarefas periódicas — Backups diários, limpeza de dados, sincronizações
  • Webhooks assíncronos — Recebe e processa callbacks de terceiros

📋 2. Criando e Gerenciando Jobs

Vá em Jobs no menu lateral para ver todos os jobs e criar novos.

📝 Criar um Job Manualmente

  1. Clique no botão + Novo Job
  2. Preencha os campos:
    • Fila: Escolha onde o job vai rodar (ex: email-sender)
    • Webhook URL: A URL que será chamada quando o job executar
    • Método HTTP: GET, POST, PUT, PATCH ou DELETE
    • Payload: Dados que serão enviados no body (JSON)
    • Callback URL (opcional): URL para receber resultado
  3. Clique em Criar Job
💡 Exemplo Prático: Consultar CEP

Você quer consultar o CEP 01001-000 na API ViaCEP:

  • Fila: cep-lookup
  • Webhook URL: https://viacep.com.br/ws/01001000/json/
  • Método: GET
  • Callback URL: https://seusistema.com.br/webhooks/cep (para receber o resultado)

O JobSync chama o ViaCEP e quando terminar, envia o resultado para seu callback.

👀 Ver Detalhes de um Job

Na lista de jobs, clique em qualquer job para ver:

  • Status atual: PENDING, PROCESSING, COMPLETED, FAILED, etc.
  • Tentativas: Quantas vezes tentou executar
  • Resultado: Resposta da API quando completou
  • Erro: Mensagem de erro se falhou
  • Logs: Histórico de cada tentativa com duração

🔄 Ações em Jobs

  • Cancelar: Para jobs PENDING/SCHEDULED — remove da fila
  • Retry: Para jobs FAILED — tenta executar de novo

📬 3. Filas (Queues)

Vá em Queues para gerenciar as filas de processamento.

🔧 O que são Filas?

Uma Fila é onde os jobs ficam aguardando processamento. Cada fila tem suas próprias configurações:

  • Concurrency: Quantos jobs podem rodar em paralelo nessa fila
  • Max Retries: Quantas vezes tentar novamente se falhar
  • Timeout: Tempo máximo de execução (ms)
  • Rate Limit: Limite de jobs por minuto (opcional)

📁 Filas Padrão do Sistema

📧 email-sender

Envio de emails em massa

💳 payment-processor

Processamento de pagamentos

📢 push-notifications

Envio de push mobile

📄 report-generator

Geração de relatórios

➕ Criar Nova Fila

  1. Clique em + Nova Fila
  2. Dê um nome (ex: webhook-receiver)
  3. Configure:
    • Concurrency: 5 (recomendado para API externas)
    • Max Retries: 3 (tenta 3 vezes se falhar)
    • Timeout: 300000 (5 minutos)
    • Rate Limit: Deixe vazio para sem limite

⏸️ Pausar e Retomar Fila

Na página de detalhes da fila:

  • Pause: Para de processar novos jobs (jobs que estão rodando terminam)
  • Resume: Volta a processar jobs da fila

⏰ 4. Jobs Recorrentes

Vá em Recorrências para automatizar tarefas que precisam rodar periodicamente.

🔄 O que são Jobs Recorrentes?

É uma tarefa que se repete automaticamente — você configura uma vez, e o JobSync dispara jobs novos automaticamente segundo sua regra.

➕ Criar Recorrência

  1. Clique em + Nova Recorrência
  2. Preencha:
    • Nome: Nome descritivo (ex: Backup diário)
    • Fila: Qual fila vai processar
    • Tipo de Recorrência:
      • Intervalo: Repete a cada X minutos
      • Cron: Expressão cron complexa (ex: 0 9 * * * = às 9h todos os dias)
    • Webhook URL: URL que será chamada
    • Payload Template: Dados enviados cada vez (JSON)
    • Máximo de Execuções: Vazio = ilimitado
    • Continuar mesmo se falhar: Se marcado, continua agendando mesmo se der erro
💡 Exemplo Prático: CEP a cada 2 minutos

Consultar ViaCEP a cada 2 minutos:

  • Nome: Consulta ViaCEP Recorrente
  • Fila: cep-lookup
  • Tipo: Intervalo
  • Intervalo: 2 (minutos)
  • Webhook URL: https://viacep.com.br/ws/01001000/json/
  • Método: GET

⚡ Trigger Manual

Quer disparar uma recorrência agora sem esperar o próximo agendamento? Clique no botão ⚡ Trigger — cria um job imediato.

✨ Feedback Enquanto processa, o botão mostra spinner. Quando termina, aparece um toast verde com o ID do job criado.

🔌 5. Integração via API

Seu sistema pode criar jobs programaticamente via REST API. Use sua API Key em Settings.

🔑 Obter API Key

  1. Vá em Settings
  2. Copie a API Key (ex: jsk_a1b2c3d4e5f6...)
  3. Use no header Authorization: Bearer jsk_a1b2c3d4e5f6...
  4. Use o header X-Organization-Id: org_demo_001

📡 Endpoints Principais

POST /api/jobs — Criar Job

POST /api/jobs 
curl -X POST https://job-assync-api.ddw1sl.easypanel.host/api/jobs \
  -H "Authorization: Bearer jsk_a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2" \
  -H "X-Organization-Id: org_demo_001" \
  -H "Content-Type: application/json" \
  -d '{
    "queue": "email-sender",
    "webhookUrl": "https://api.seusistema.com.br/send-email",
    "webhookMethod": "POST",
    "payload": {
      "to": "cliente@email.com",
      "subject": "Bem-vindo!",
      "body": "Olá! Bem-vindo ao sistema."
    }
  }'

Resposta:

{
  "id": "cmmu9av520009fny85gruxxp2",
  "status": "PENDING",
  "queue": "email-sender",
  "createdAt": "2026-03-17T07:15:00.000Z"
}

GET /api/jobs — Listar Jobs

GET /api/jobs?limit=20&status=COMPLETED 
curl https://job-assync-api.ddw1sl.easypanel.host/api/jobs?limit=20 \
  -H "Authorization: Bearer jsk_a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2" \
  -H "X-Organization-Id: org_demo_001"

Filtros:

  • status: PENDING, PROCESSING, COMPLETED, FAILED, etc.
  • queue: Nome da fila
  • limit: Quantos resultados (padrão: 20)

DELETE /api/jobs/{id}

  -H "X-Organization-Id: org_demo_001"

Cancela jobs PENDING ou SCHEDULED (remove da fila).

⚠️ Importante: Webhook URL

A Webhook URL que você configura no job é a URL que o JobSync vai chamar quando o job executar. Geralmente é uma URL do seu sistema que recebe e processa o resultado.

Fluxo completo:

  1. Seu sistema cria job no JobSync → JobSync coloca na fila
  2. JobSync executa o job → Chama sua Webhook URL com o payload
  3. Seu sistema processa → Retorna resposta
  4. JobSync salva resultado → Job fica COMPLETED

📝 Exemplo de Integração (Node.js)

import axios from 'axios';

const API_URL = 'https://job-assync-api.ddw1sl.easypanel.host';
const API_KEY = 'jsk_a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2';
const ORG_ID = 'org_demo_001';

// Criar job para enviar email
async function enviarEmail(destinatario, assunto, corpo) {
  const response = await axios.post(`${API_URL}/api/jobs`, {
    queue: 'email-sender',
    webhookUrl: 'https://api.seusistema.com.br/send-email',
    webhookMethod: 'POST',
    payload: {
      to: destinatario,
      subject: assunto,
      body: corpo
    }
  }, {
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
      'X-Organization-Id': ORG_ID
    }
  });

  return response.data; // { id, status, queue, createdAt }
}

// Consultar status depois
async function consultarStatus(jobId) {
  const response = await axios.get(`${API_URL}/api/jobs/${jobId}`, {
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
      'X-Organization-Id': ORG_ID
    }
  });

  return response.data; // { id, status, result, error, logs }
}

// Exemplo de uso
const job = await enviarEmail(
  'cliente@email.com',
  'Bem-vindo!',
  'Olá! Bem-vindo ao nosso sistema.'
);

console.log('Job criado:', job.id);

// Depois verificar se completou
setTimeout(async () => {
  const status = await consultarStatus(job.id);
  console.log('Status:', status.status);
  console.log('Resultado:', status.result);
}, 5000);

🎯 Casos de Uso Reais

1. E-commerce: Envio de Email após Compra

Quando cliente finaliza pedido:

  1. Seu sistema cria job no JobSync com dados do pedido
  2. JobSync processa em fila (não trava o checkout)
  3. Job chama seu webhook → seu sistema envia email
  4. Job salva resultado → você pode consultar depois

2. SaaS: Relatório Mensal para Usuário

Todo dia 1 do mês às 9h:

  1. Job recorrente dispara automaticamente
  2. Job chama seu webhook → seu sistema gera PDF
  3. Seu sistema envia email com anexo
  4. Job fica COMPLETED com resultado salvo

3. API Gateway: Sincronização com CRM Externo

Sempre que usuário atualiza perfil:

  1. Seu sistema cria job para sincronizar com CRM
  2. Job chama webhook → seu sistema faz PUT na API do CRM
  3. Se falhar, JobSync tenta mais 2x automaticamente
  4. Se falhar tudo, você recebe alerta e pode retry manual

❓ Perguntas Frequentes

❓ Qual a diferença entre PENDING, PROCESSING e COMPLETED?

  • PENDING: Job foi criado, está na fila aguardando
  • PROCESSING: Job está rodando agora
  • COMPLETED: Job terminou com sucesso
  • FAILED: Job falhou (mas pode ter retry)
  • DEAD: Job falhou todas as tentativas possíveis

❓ O que acontece se um job falhar?

O JobSync tenta novamente automaticamente (configurado na fila, padrão: 3 tentativas). Se falhar tudo, vira DEAD. Você pode clicar em Retry para tentar de novo.

❓ Posso editar um job depois de criado?

Não. Jobs são imutáveis depois de criados. Mas você pode cancelar e criar um novo, ou usar Retry para executar novamente.

❓ Como fazer jobs que dependem de outros jobs?

Crie jobs em sequência: quando o primeiro completar, seu callback cria o próximo job. Ou use workflows complexos no código (não implementado no dashboard ainda).

❓ Tem limite de jobs?

Não. Você pode criar milhões de jobs. O sistema processa em filas paralelas, então escala automaticamente com a carga.
