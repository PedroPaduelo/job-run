6. Gerenciamento de Estado

O JobSync utiliza uma estratégia híbrida de gerenciamento de estado, combinando TanStack Query para estado servidor e React useState para estado cliente.

🖥️

Estado Servidor

TanStack Query - Cache, refetch automático, paginação

  • staleTime: 1000ms
  • retry: 1
  • Custom hooks: useJobs, useQueues, useStats, useApiKeys
Server State
🎯

Estado Cliente

React useState - Forms, toggles, UI state

  • Filtros de busca
  • Modais e dialogs
  • Estado de loading local
Client State
🔐

Autenticação

@nommand/auth-sdk - OAuth 2.0

  • User + Token
  • Lista de organizações
  • Permissões
  • Refresh automático de token

7. Sistema de Rotas

O roteamento é feito com React Router v7. Todas as rotas (exceto /login e /callback) são protegidas por um wrapper de autenticação.

Mapa de Rotas

Rota Componente Protegida
/ Dashboard Sim
/jobs Jobs Sim
/jobs/:id JobDetail Sim
/queues Queues Sim
/queues/:id QueueDetail Sim
/settings Settings Sim
/login Login Não
/callback Callback Não
ℹ️
ProtectedRoute

O componente ProtectedRoute envolve todas as rotas autenticadas. Ele verifica o status de autenticação e redireciona para /login se necessário.

8. Cliente API

O frontend utiliza um cliente fetch customizado em vez de Axios. Toda a comunicação HTTP passa por src/lib/api.ts.

Características do Cliente API

  • Token injection: Automaticamente injeta o token de acesso (nommand_access_token ou token)
  • Header X-Organization-Id: Inclui o ID da organização selecionada
  • Error handling: Lança erro em responses não-ok
  • URL base: VITE_API_URL (padrão: http://localhost:3001)

Exemplo de Uso

// GET /api/jobs
const { jobs, total } = await jobsApi.list({ status: 'FAILED', limit: 10 });

// GET /api/jobs/:id
const job = await jobsApi.get('job-123');

// POST /api/jobs
const newJob = await jobsApi.create({
  queue: 'my-queue',
  webhookUrl: 'https://example.com/webhook',
  payload: { key: 'value' }
});

// GET /api/queues
const queues = await queuesApi.list();

10. Fluxo de Autenticação

O JobSync utiliza OAuth 2.0 com o @nommand/auth-sdk. O AuthContext combina a SDK com contexto de organização local.

┌─────────────────┐ ┌──────────────────┐ ┌─────────────────┐ │ /login │ ───► │ Auth Service │ ───► │ /callback │ │ (redirect) │ │ (OAuth) │ │ (token exchange) │ └─────────────────┘ └──────────────────┘ └─────────────────┘ │ ▼ ┌─────────────────────────┐ │ AuthContext │ │ - user │ │ - token │ │ - organization │ │ - organizations list │ └─────────────────────────┘

AuthContext - Provider Hierarchy

ThemeProvider (defaultTheme="system")
└── AuthProvider (@nommand/auth-sdk)
    └── OrganizationProvider
        └── QueryClientProvider
            └── BrowserRouter
                └── Routes
                    ├── /login (público)
                    ├── /callback (público)
                    └── /* (protegido via ProtectedRoute)
                        └── Layout
                            └── Routes aninhadas
