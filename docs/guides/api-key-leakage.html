API Key Leakage

Vazamento de chaves em logs e mensagens de erro

🔑 O Problema

API Keys são usadas como Bearer token no header Authorization. Se logs ou error messages incluírem o header, a key vaza.

🚨
Credential Exposure

Fastify logger, Sentry, stack traces podem capturar headers, expondo a API key em logs.

📊 Código Vulnerável

// server.ts onRequest (linha 71)
fastify.addHook('onRequest', async (request, reply) => {
  const authHeader = request.headers.authorization;
  // ...
});

request.headers.authorization pode ser logado por:

  • Fastify default logger (se log level = info)
  • Sentry captureError(request, error) — request object pode incluir headers
  • Erros manualmente logados: fastify.log.error({ req: request }, 'something')

💥 Vetores de Vazamento

  • Logfiles — arquivos de log em disco podem conter Bearer tokens
  • Sentry/APM — headers IncludedByDefault
  • Error responses — se erro retorna request details (debug mode)
  • Audit Log — não deve salvar a key, mas log de autenticaçãofalha pode ter IP + user-agent, não a key em si (ok)

✅ Recomendações

  1. Sanitizar headers nos logs — configure Fastify logger: 
    fastify.log = Logger.getInstance({
  transport: {
    target: 'pino-pretty',
    options: {
      hidePayload: true,
      ignore: 'req,res,req.headers.authorization'
    }
  }
});
  2. Sentry beforeSend — filtrar headers.authorization antes de enviar: 
    Sentry.init({
  beforeSend(event) {
    if (event.request?.headers?.authorization) {
      event.request.headers.authorization = 'Bearer ***REDACTED***';
    }
    return event;
  }
});
  3. Nunca retornar headers em erros — error handler NÃO deve incluir request.headers
  4. Rotacionar API Keys periodicamente — exp: 90 dias
  5. Revoke immediately on suspicion — endpoint para revocar imediatamente
  6. Audit de uso — log de criação/revogação, mas nunca a key em si

🔍 Checklist

  • Fastify logger ignora authorization header? ❌
  • Sentry beforeSend redacta headers? ❌
  • Error responses incluem request? ⚠️ (dev-only)
  • API keys têm expiração? ✅ (expiresAt field exists)
  • Há endpoint de revogação instantânea? ✅ (POST /apikeys/:id/revoke)
