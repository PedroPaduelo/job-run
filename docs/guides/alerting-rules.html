Alerting Rules

Thresholds para PagerDuty/Opsgenie

🚨 Quando Alertar

MetricThresholdSeveridadeAção
Jobs pendentes > 1000 por 5minP1Investigar fila parada
Failed job rate > 5% (última hora)P2Verificar webhooks down
Worker count ↓ (um worker morre)P1Auto-restart
Redis memória > 85%P2Cleanup keys
DB connections > 80% poolP2Aumentar pool ou otimizar queries
API latency p95 > 2sP3Investigar gargalo

✅ Recomendações

  1. Configurar alerts no Grafana/Prometheus usando query ranges
  2. Usar multi-channel escalation (Slack → PagerDuty se >30min)
  3. Runbook anexo em cada alert (link para troubleshooting)

🔍 Checklist

  • Thresholds definidos? ❌
  • Alertas implementados? ❌
  • Runbooks linkados? ❌
  • Testes de alerta? ❌
