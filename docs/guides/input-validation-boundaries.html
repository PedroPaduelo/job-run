Input Validation Boundaries

Onde validar, schemas Zod, defesa em profundidade

📋 Camadas de Validação

// 1. HTTP Request → Fastify body/query
fastify.post('/jobs', async (request, reply) => {
  const result = createJobSchema.safeParse(request.body);  // Zod
  if (!result.success) {
    return reply.status(400).send({ error: 'Validation error', details: result.error.flatten() });
  }
  // ... continua para service
});

// 2. Service → Database
const job = await prisma.job.create({ data: {...} }); // Prisma valida schema

📊 Schemas Zod Atuais

// src/types/index.ts

// Job
createJobSchema = z.object({
  queue: z.string().min(1),
  webhookUrl: z.string().url(),       // ← URL validada
  webhookMethod: z.enum(['GET','POST','PUT','PATCH','DELETE']),
  callbackUrl: z.string().url().optional(),
  payload: z.record(z.unknown()),    // ← JSON qualquer
  userId: z.string().optional(),
  externalId: z.string().optional(),
  priority: z.number().int().min(0).max(100),
  scheduledAt: z.string().datetime().optional(),
  metadata: z.record(z.unknown()).optional(),
});

// Queue
createQueueSchema = z.object({
  name: z.string().min(1).max(100),
  concurrency: z.number().int().min(1).max(100).default(5),
  maxRetries: z.number().int().min(0).max(10).default(3),
  retryDelay: z.number().int().min(1000).max(3600000).default(5000),
  timeout: z.number().int().min(5000).max(3600000).default(300000),
  rateLimit: z.number().int().min(1).optional(),
});

✅ Boas Práticas Implementadas

  • Zod.safeParse() — não lança exceção, retorna { success, data, error }
  • Coerce tipos — query params chegam como string, z.coerce.number() converte
  • Defaults sensatos — timeout = 5min, maxRetries = 3
  • Limites de tamanho — max 100 chars em name, max 100 retries

⚠️ Gaps de Validação

  • webhookUrl sem SSRF check — pode ser localhost, 169.254.169.254
  • callbackUrl sem SSRF check — mesmo problema
  • payload sem tamanho max — pode ser GBs de JSON
  • metadata sem tamanho max — mesmo problema
  • bulk jobs max 1000 — bom limite, mas não configurável

✅ Recomendações

  1. SSRF Validation — adicionar em createJobSchema: 
    const webhookUrlSchema = z.string().url().refine(
  (url) => !isPrivateURL(url),
  { message: "Webhook URL cannot point to private/internal addresses" }
);
  2. Payload/Metadata size limit: 
    payload: z.record(z.unknown()).max(1024 * 100), // 100KB max
  3. URL scheme validation — só https em produção: 
    webhookUrl: z.string().url().refine(
  (url) => process.env.NODE_ENV === 'production' ? url.startsWith('https') : true
)
  4. Rate limit validation — range menor, evitar 0
  5. Input sanitization — se webhookUrl é refletido em logs, sanitizar

🔍 Checklist

  • Zod validation? ✅
  • Prisma schema validation? ✅
  • URL SSRF check? ❌
  • Payload size limit? ❌
  • HTTPS only em prod? ⚠️
  • Input sanitized em logs? ⚠️
