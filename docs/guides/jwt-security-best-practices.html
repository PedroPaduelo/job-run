JWT Security Best Practices

Autenticação JWT: algoritmo, expiração, validação

🔐 Implementação Atual

// src/lib/auth.ts
await verifyJWT(token) // qual algoritmo? Qual validação?
await getUserOrganizations(token)
await buildAuthenticatedUser(token, decoded, orgId)

Sem acesso ao código de auth.ts no momento, mas presume-se:

  • Usa JWT do Auth Service externo (não local)
  • Extrai organizationId do token ou via callback
  • Valida assinatura com chave pública (RS256)
ℹ️
Arquitetura

JWT é fornecido por Auth Service externo. JobSync apenas valida, não emite tokens.

✅ Best Practices a Verificar

1. Algoritmo

  • RS256 ou ES256 — assinatura com chave privada, verificação com pública
  • NUNCA usar none — bypass de assinatura
  • NUNCA usar HS256 com segredo compartilhado em produção (vazamento = forge tokens)

2. Expiração

  • exp claim — verificar expiração
  • Access token: 15min - 1h
  • Refresh token: dias-semanas (via Auth Service)

3. Validação

  • iss — issuer esperado (Auth Service URL)
  • aud — audience esperada (job-sync)
  • sub — subject (user ID)
  • iat — não antes de agora

4. Headers

  • typ: JWT — verificar tipo
  • alg — não confiar cego, verificar whitelist

💥 Vulnerabilidades

  • Algorithm Confusion — attacker muda alg para "none", servidor aceita
  • Key Confusion — RS256 → HS256 usando chave pública como segredo HMAC
  • Expired tokens — se verificação de exp está desabilitada
  • Weak secret — se HS256 usado com secret curto

✅ Recomendações

  1. Verificar implementação de verifyJWT() — garantir que: 
    jwt.verify(token, publicKey, {
  algorithms: ['RS256', 'ES256'],  // whitelist
  issuer: process.env.JWT_ISSUER,
  audience: process.env.JWT_AUDIENCE,
  clockTolerance: 30               // 30s tolerance para clock skew
});
  2. Logging de token validation failures — para detectar ataques
  3. Token revocation — se JWT é stolen, como revoke? ( blacklist em Redis?)
  4. Rate limit em /auth — previne brute force de tokens

🔍 Checklist

  • Algorithms whitelist (RS256/ES256)? ⚠️
  • Issuer validation? ⚠️
  • Audience validation? ⚠️
  • Expiration validation? ⚠️
  • Clock tolerance configurado? ❌
  • Token blacklist/revocation? ❌
