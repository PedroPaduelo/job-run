Chaos Engineering

Experimentos: kill Redis, slow DB, network latency

🎯 Objetivo

Testar resiliência do sistema sob condições adversas.

🧪 Experimentos

ExperimentoImpactoResultado Esperado
Kill RedisAltaWorkers falham, reconnect, jobs retry
DB latency +500msMédiaAPI responses lentas, timeout bubbles
Worker crash randomMédiaPM2 restart, jobs reassignados
Network partitionAltaPartial outage, circuit breaker

✅ Recomendações

  1. Começar small — matar um worker por vez
  2. Blast radius control — não afetar produção inteira
  3. Runbook atualizado — saber como recuperar
  4. Post-mortem — documentar lições aprendidas

🔍 Checklist

  • Chaos tool configurado? ❌
  • Experiments planejados? ❌
  • Blast radius control? ❌
  • Post-mortem process? ⚠️
