🎣 Custom Hooks

Hooks que encapsulam React Query e lógica específica do domínio.

useJobs(filters)

const { data, isLoading, error, refetch } = useJobs({
  status?: 'completed' | 'failed' | 'processing',
  queue?: string,
  from?: Date,
  to?: Date,
  limit?: number,
  offset?: number
})

const { mutate: createJob } = useCreateJob()
const { mutate: cancelJob } = useCancelJob()
const { mutate: retryJob } = useRetryJob()
  • Refetch automático a cada 30s
  • Invalidates queries relacionadas nas mutations

useJob(id)

const { data, isLoading } = useJob(jobId)

// Ações integradas:
cancelJob({ jobId })
retryJob({ jobId })
  • Refetch a cada 5s se job está ativo
  • Perfect para JobDetail page

useQueues() / useQueue(id)

const { data: queues } = useQueues()
const { data: queue } = useQueue(queueId)
const { data: stats } = useQueueStats(queueId)

const { mutate: createQueue } = useCreateQueue()
const { mutate: updateQueue } = useUpdateQueue()
const { mutate: deleteQueue } = useDeleteQueue()
const { mutate: pauseQueue } = usePauseQueue()
const { mutate: resumeQueue } = useResumeQueue()
  • Queues list refetch 30s
  • Queue stats refetch 10s

useStats() / useTimeline(hours)

const { data: overview } = useStats()
const { data: timeline } = useTimeline(24) // última 24h

// overview: { totalJobs, todayJobs, processing, failedToday, avgDuration, queues[] }
  • Overview refetch 30s
  • Timeline refetch 60s (overview page)

useApiKeys()

const { data: apiKeys } = useApiKeys()

const { mutate: createApiKey } = useCreateApiKey()
const { mutate: updateApiKey } = useUpdateApiKey()
const { mutate: deleteApiKey } = useDeleteApiKey()
const { mutate: revokeApiKey } = useRevokeApiKey()
  • Refetch a cada 30s
  • Create mostra key apenas uma vez

useDebounce(value, delay)

const [search, setSearch] = useState('')
const debouncedSearch = useDebounce(search, 300)

// debouncedSearch só atualiza após 300ms de inatividade
  • Usado no filtro de jobs
  • Evita refetch excessivo

Toggle Button

import { useTheme } from '@nomad/ui'

function ThemeToggle() {
  const { theme, setTheme } = useTheme()
  return (
    <Button onClick={() =>
      setTheme(theme === 'dark' ? 'light' : 'dark')
    }>
      {theme === 'dark' ? '☀️' : '🌙'}
    </Button>
  )
}

Theme persistido em localStorage.

🧠 Architecture Patterns

Data Flow

API Layer (lib/api.ts)
    ↓ fetch wrapper com token/org headers
React Query (QueryClient)
    ↓ cache + auto-refetch
Page Components (pages/)
    ↓ mutations invalidate queries
UI Updates (reactive)

Server-state: React Query. Client-state: Contexts (Auth, Org).

Folder Structure

frontend-job-async/
├── src/
│   ├── components/         # Layout, ProtectedRoute
│   ├── contexts/           # AuthContext
│   ├── hooks/              # useJobs, useQueues, etc.
│   ├── lib/                # api.ts, constants.ts, toast.ts
│   ├── pages/              # 8 páginas roteadas
│   ├── App.tsx             # Router + QueryProvider
│   └── main.tsx            # Entry + Sentry init

Authentication Flow

  1. Usuário clica "Sign in" em /login
  2. sdkAuth.authorize() redireciona para Auth Service (OAuth PKCE)
  3. Auth service redireciona de volta para /callback?code=...
  4. handleCallback() troca code por token via SDK
  5. Token salvo em localStorage (`nommand_access_token`)
  6. Organization salva em localStorage
  7. Redireciona para /
  8. Todas as requisições incluem Authorization: Bearer <token> e X-Organization-Id

ProtectedRoute

// Verifica autenticação antes de renderizar páginas
if (loading) return <Spinner />
if (!isAuthenticated) navigate('/login')
return <Outlet />

Aplica-se a todas as rotas filhas de /.
