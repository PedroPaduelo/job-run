Página de Filas (Queues)

Documentação completa da interface de gerenciamento de filas no frontend JobSync

Visão Geral

A página de Filas (Queues) é um componente fundamental do painel administrativo do JobSync. Ela permite que usuários visualizem, criem, pausem, retomem e excluam filas de processamento de jobs. Esta página serve como ponto central para o gerenciamento de todas as filas configuradas na organização.

ℹ️
Localização do Componente

O código fonte está localizado em: frontend-job-async/src/pages/Queues.tsx

Layout da Página

A página de Filas possui uma estrutura organizada em três áreas principais:

2.1 Cabeçalho

O cabeçalho contém o título da página e o botão de criação de novas filas:

  • Título: "Queues" - Exibido em texto grande com destaque
  • Subtítulo: "Manage your job queues" - Texto de descrição em tom mais suave
  • Botão "Create Queue": Botão principal que abre o formulário de criação

2.2 Formulário de Criação de Fila

Ao clicar em "Create Queue", um cartão (Card) é exibido inline com os seguintes campos:

CampoTipoPadrãoDescrição
Queue NameInput (texto)—Nome identificador da fila (obrigatório)
ConcurrencyInput (número)5Número de jobs processados simultaneamente (1-100)
Max RetriesInput (número)3Tentativas máximas em caso de falha (0-10)
TimeoutInput (número)300000ms (5 min)Tempo máximo de execução em segundos (1-3600)
Rate LimitInput (número)0 (ilimitado)Limite de jobs por minuto (0 = sem limite)
ℹ️
Nota sobre Timeout

O campo Timeout é armazenado em milissegundos internamente, mas exibido em segundos na interface para melhor usabilidade.

2.3 Tabela de Filas

A tabela principal exibe todas as filas cadastradas com as seguintes colunas:

ColunaDescrição
NameNome da fila (linkável para detalhes)
ConcurrencyValor numérico da concorrência
Max RetriesNúmero máximo de tentativas
TimeoutTempo limite formatado (ex: "300s")
Rate LimitFormato: "X/min" ou "None" se ilimitado
StatusBadges de estado (Active/Inactive + Paused)
ActionsBotões de ação: Visualizar, Pausar, Retomar, Excluir

Validação

O campo Queue Name é obrigatório. A validação ocorre no cliente antes do envio:

if (!newQueue.name.trim()) {
  toast.error("Queue name is required");
  return;
}

Hooks Envolvidos

A criação de filas utiliza o hook useCreateQueue():

export function useCreateQueue() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: queuesApi.create,
    onSuccess: () => {
      // Invalida cache para forçar refetch
      queryClient.invalidateQueries({ queryKey: ["queues"] });
    },
  });
}

Ações de Gerenciamento

A página oferece quatro ações principais para cada fila:

3.1 Pausar Fila (Pause)

Pausa temporariamente o processamento de jobs na fila:

  • Endpoint: POST /queues/:id/pause
  • Efeito: Define isPaused = true
  • Hook: usePauseQueue()
  • Feedback: Badge "Paused" aparece na tabela

3.2 Retomar Fila (Resume)

Retoma o processamento de jobs em uma fila pausada:

  • Endpoint: POST /queues/:id/resume
  • Efeito: Define isPaused = false
  • Hook: useResumeQueue()
  • Feedback: Badge "Paused" é removido

3.3 Excluir Fila (Delete)

Realiza um soft-delete da fila (não remove fisicamente do banco):

  • Endpoint: DELETE /queues/:id
  • Efeito: Define isActive = false
  • Hook: useDeleteQueue()
  • Confirmação: Usa confirm() nativo do navegador
⚠️
Soft-Delete

A exclusão de filas é um soft-delete. A fila permanece no banco de dados mas é marcada como inativa, deixando de ser descoberta pelos workers.

Código de Ações

// Deletar
const handleDelete = async (id: string) => {
  if (confirm("Are you sure you want to deactivate this queue?")) {
    await deleteQueue.mutateAsync(id);
  }
};

// Pausar
const handlePause = async (id: string) => {
  await pauseQueue.mutateAsync(id);
};

// Retomar
const handleResume = async (id: string) => {
  await resumeQueue.mutateAsync(id);
};

Indicadores de Status

A coluna de Status exibe badges coloridos que indicam o estado atual da fila:

BadgeCorCondição
ActiveVerde (success)isActive === true
InactiveCinza (secondary)isActive === false
PausedAmarelo (warning)isPaused === true
ℹ️
Combinação de Badges

Uma fila pode exibir múltiplos badges simultaneamente. Por exemplo: "Active" + "Paused" indica que a fila está ativa mas temporariamente pausada.

Estado de Carregamento

Durante a execução de mutações (criação, pausa, resume, delete), os botões são desabilitados para evitar ações duplicadas:

<Button
  onClick={() => handlePause(queue.id)}
  disabled={pauseQueue.isPending}  // Bloqueia durante mutation
  title="Pause queue"
>
  <Pause className="h-4 w-4" />
</Button>

Navegação da Tabela

O nome de cada fila na tabela é um link que redireciona para a página de detalhes:

  • Link: /queues/:id
  • Ícone: Olho (Eye) na coluna Ações
  • Componente: <Link to={`/queues/${queue.id}`}>

Ambas as células (nome e ícone de olho) redirecionam para a mesma página de detalhes da fila, permitindo visualizar estatísticas, jobs e configurações detalhadas.

Estado Vazio (Empty State)

Quando não existem filas cadastradas, uma mensagem amigável é exibida:

"No queues found. Create one to get started."

Esta mensagem aparece na tabela quando:

  • queues é undefined ou null
  • O array queues está vazio (length === 0)

Hooks Personalizados

A página de Filas utiliza vários hooks do TanStack Query para gerenciar o estado e as requisições:

5.1 useQueues()

Busca a lista de todas as filas da organização:

export function useQueues() {
  return useQuery({
    queryKey: ["queues"],
    queryFn: queuesApi.list,
    refetchInterval: REFETCH_INTERVALS.QUEUES,  // Auto-refresh
    enabled: isAuthenticated,
  });
}

5.2 useCreateQueue()

Cria uma nova fila:

export function useCreateQueue() {
  return useMutation({
    mutationFn: queuesApi.create,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["queues"] });
    },
  });
}

5.3 useDeleteQueue()

Exclui (soft-delete) uma fila:

export function useDeleteQueue() {
  return useMutation({
    mutationFn: queuesApi.delete,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["queues"] });
    },
  });
}

5.4 usePauseQueue()

Pausa o processamento de uma fila:

export function usePauseQueue() {
  return useMutation({
    mutationFn: queuesApi.pause,
    onSuccess: (_data, id) => {
      queryClient.invalidateQueries({ queryKey: ["queues"] });
      queryClient.invalidateQueries({ queryKey: ["queue", id] });
    },
  });
}

5.5 useResumeQueue()

Retoma o processamento de uma fila pausada:

export function useResumeQueue() {
  return useMutation({
    mutationFn: queuesApi.resume,
    onSuccess: (_data, id) => {
      queryClient.invalidateQueries({ queryKey: ["queues"] });
      queryClient.invalidateQueries({ queryKey: ["queue", id] });
    },
  });
}
ℹ️
Localização dos Hooks

Todos os hooks estão implementados em: frontend-job-async/src/hooks/useQueues.ts

Permissões e Autorização

O sistema de permissões é implementado no backend, mas o frontend deve renderizar os botões condicionalmente baseado nas permissões do usuário:

PermissãoDescriçãoBotões Afetados
queues:createCriar novas filasBotão "Create Queue"
queues:deleteExcluir filasBotão de lixeira
queues:pausePausar/Retomar filasBotões Pause/Resume
ℹ️
Implementação de Permissões

A implementação de renderização condicional baseada em permissões pode variar. O código atual exibe todos os botões, mas a camada de permissão no backend garante que apenas usuários autorizados possam executar essas ações.

Fluxo Typical UI

A seguir, apresentamos o fluxo típico de uso da página de Filas:

1. Visualização Inicial

  1. Usuário acessa a página /queues
  2. Hook useQueues() busca automaticamente a lista de filas
  3. Enquanto carrega, exibe "Loading..."
  4. Após carregamento, exibe a tabela com as filas ou estado vazio

2. Criar uma Nova Fila

  1. Usuário clica no botão "Create Queue" no cabeçalho
  2. Formulário inline expande abaixo do cabeçalho
  3. Usuário preenche os campos (nome é obrigatório)
  4. Ao submeter, valida o nome
  5. Se válido, chama createQueue.mutateAsync()
  6. Em sucesso: exibe toast, limpa formulário, fecha card, atualiza tabela
  7. Em erro: exibe toast com mensagem de erro

3. Gerenciar Filas Existentes

  1. Usuário visualiza a lista de filas na tabela
  2. Pode clicar no nome ou ícone de olho para ver detalhes
  3. Pode clicar em Pause para pausar uma fila ativa
  4. Pode clicar em Resume para retomar uma fila pausada
  5. Pode clicar em Delete para excluir (com confirmação)
  6. Todas as ações mostram feedback visual durante o carregamento

4. Estados Visuais Durante Ações

⏳

Carregando

Botões desabilitados com estado isPending

✓

Sucesso

Toast de sucesso + tabela atualizada automaticamente

✕

Erro

Toast de erro com mensagem da API
