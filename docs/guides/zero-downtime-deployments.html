Zero-Downtime Deployments

Blue-green, rolling update, migrações

🔄 Estratégias

  • Blue-Green: deploy nova versão em paralelo, switchear traffic via load balancer
  • Rolling Update: substitui pods/containers gradualmente
  • Canary: 5% do tráfego vai pra nova versão, monitora, escala

📋 Passo-a-passo (Kubernetes)

1. Build Docker image: jobsync:2.0.0
2. push to registry
3. kubectl set image deployment/jobsync jobsync=...:2.0.0 --record
4. kubectl rollout status deployment/jobsync  # monitorar
5. Se rollback necessário:
   kubectl rollout undo deployment/jobsync

⚠️ Database Migrations

Prisma migrations devem ser backward-compatible:

  • Não dropar columns usados pela versão antiga
  • Adicionar columns opcionais primeiro, depois preencher, depois NOT NULL
  • Rollback automatizado via prisma migrate deploy

✅ Recomendações

  1. Health checks no /ready (Redis + DB)
  2. Rolling update com maxUnavailable=0 (garante zero downtime)
  3. Feature flags para toggle rápido
  4. Auto-rollback em métricas de erro (Sentry + PagerDuty)

🔍 Checklist

  • Health checks /ready? ✅
  • Rolling update configurado? ⚠️
  • Migrations backward-compatible? ⚠️
  • Rollback testado? ❌
  • Feature flags? ❌
