Capacity Planning

Projeção de carga, SLOs, dimensionamento

📈 Métricas Chave

  • Jobs por segundo (throughput)
  • Latência p95 (tempo até job ser aceito)
  • Taxa de falha (jobs com status FAILED/DEAD)
  • Concorrência ativa (workers processando simultâneo)

🎯 SLOs Sugeridos

MetricTargetSeveridade
Availability99.9%Alta
Job acceptance latency< 100ms p95Média
Job completion rate (24h)> 99.5%Alta
Failed job rate< 1%Média

📊 Dimensionamento

Cenário: 1000 jobs/min = ~17 jobs/sec
Concorrência média por job: 2s (webhook)
 → Workers necessários: 17 * 2 = 34
Com margem 50%: 50 workers

CPU: 1 worker ~ 50MB RAM + 0.1 CPU
50 workers → 2.5GB RAM + 5 CPU

✅ Recomendações

  1. Load test — simular 10k jobs, medir saturação
  2. Auto-scaling — baseado em queue.waiting count
  3. Capacity buffer — 30% extra para spikes
  4. Monitorar saturation — queue.active vs concurrency max

🔍 Checklist

  • SLOs definidos? ❌
  • Load test realizado? ❌
  • Auto-scaling config? ❌
  • Capacity buffer adequado? ❌
