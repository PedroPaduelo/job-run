Load Testing Approach

k6, locust, 10k jobs/s, saturação de workers

🎯 Objetivos

  • Identificar bottlenecks (DB, Redis, CPU)
  • Medir throughput máximo sustentável
  • Validar concorrência de workers
  • Testar recovery de falhas

📊 Ferramentas

  • k6 — load test HTTP API (criar jobs)
  • Locust — Python, bom para webhooks customizados
  • Autocannon — Node.js, simples

📈 Cenários

CenárioLoadDuração
Smoke100 req/s5min
Normal1000 req/s30min
Stress5000 req/s15min
Soak500 req/s24h

✅ Recomendações

  1. Ambiente isolado — não testar em produção
  2. Seed data — pré-carregar queues, organizações
  3. Métricas coletadas — latency, error rate, DB pool, Redis mem
  4. Auto-scale validation — testar se scaling funciona
  5. Report — documento com resultados e capacity planning

🔍 Checklist

  • Ferramenta escolhida? ❌
  • Ambiente de testeReady? ❌
  • Métricas coletadas? ❌
  • Report template? ❌
