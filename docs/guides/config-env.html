Configuração de Ambiente (ENV)

Variáveis de ambiente essenciais do JobSync e seus valores padrão, exemplos e sensibilidade.

1. Visão geral

JobSync usa variáveis de ambiente no backend e frontend. A fonte principal de valores é .env.example (backend/front-end). Os valores de tempo de execução podem ser alterados em plataformas de deploy (Heroku, Railway, Render, Kubernetes, etc).

  • Backend: backend-job-assync/.env.example
  • Frontend: frontend-job-async/.env.example

A arquitetura suporta multi-tenant via x-organization-id e JWT, mas a configuração de ambiente em si não varia por tenant.

Como o backend usa essas variáveis

  • fastify.listen({ port: parseInt(process.env.PORT || '3001', 10) })
  • ALLOWED_ORIGINS parseado e testado no plugin CORS
  • rateLimit configurado com max: 1000 e timeWindow: '1 minute' (limite por org ou IP)
  • AUTH_SERVER_URL determina JWKS_URL para JWT
  • REDIS_URL inicializa Redis com maxRetriesPerRequest: null

4. Detalhes de segurança e sensibilidade

Classificação de sensibilidade:

  • Segredos sensíveis: DATABASE_URL (inclui credencial), REDIS_URL (se incluir senha), SENTRY_DSN, credenciais do Auth Service se usadas em outros ambientes.
  • Não sensíveis: PORT, NODE_ENV, ALLOWED_ORIGINS, WORKER_CONCURRENCY, VITE_API_URL, VITE_AUTH_URL, VITE_AUTH_CLIENT_ID.

Mantenha arquivos .env fora do controle de versão e use secret manager no deploy.

5. Multi-tenant e valores de ambiente

O JobSync é multi-tenant via payload JWT + cabeçalhos (x-organization-id) e não exige variáveis diferentes por tenant. A única variável relacionada a Auth (issuer / base URL) é global:

  • AUTH_SERVER_URL: todo tenant valida JWT contra o mesmo Auth Service.

Se você usar provedores de token diferentes, atualize AUTH_SERVER_URL apropriadamente e garanta seus iss nas rotas.

6. .env.example - comparação backend e frontend

Configurações base usadas em desenvolvimento para iniciar localmente.

Backend

# Database
DATABASE_URL=postgres://user:password@localhost:5432/job_assync

# Redis
REDIS_URL=redis://localhost:6379

# Auth Service
AUTH_SERVER_URL=http://localhost:3000

# API Server
PORT=3001
NODE_ENV=development

# CORS
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000

# Worker
WORKER_CONCURRENCY=10

# Sentry
SENTRY_DSN=

Frontend

# API URLs
VITE_API_URL=http://localhost:3001
VITE_AUTH_URL=http://localhost:3000

# Auth Client
VITE_AUTH_CLIENT_ID=job-sync

# Sentry
VITE_SENTRY_DSN=

7. Boas práticas rápidas

  • Use variáveis de ambiente separadas por ambiente (dev/staging/prod) e mantenha segredos em Secret Store.
  • Verifique valores de NODE_ENV para modo de produção.
  • Não exponha DATABASE_URL ou REDIS_URL em logs.
  • Defina ALLOWED_ORIGINS com URLs da aplicação para segurança de CORS.
  • Monitore SENTRY_DSN e configure apenas quando habilitado.
