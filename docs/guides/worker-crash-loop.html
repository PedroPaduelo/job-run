Worker Crash Loop

Worker sem restart automático causa perda de processamento

💥 O Problema

Worker rodando com tsx watch ou node dist/worker.js sem processo gerenciador (PM2/Systemd) pode crashar e não restartar.

🔁
No Auto-Restart

Se worker levanta exception não tratada, processo morre. Jobs pendentes ficam presos na fila BullMQ até que workers voltem.

📊 Código de Inicialização

// package.json scripts
"dev:worker": "tsx watch src/worker.ts"

// Em produção, como é startado?
// Se for: node dist/worker.js → sem restart automático

BullMQ não auto-reconnect se worker morre. Workers são independentes.

💥 Cenários de Crash

  • Unhandled promise rejection em processJob
  • Out of memory (processa payload gigante)
  • Redis connection lost → worker wrestling ao reconnectar
  • Database outage → prisma.$connect falha
  • SIGKILL (OOM killer)

✅ Recomendações

  1. Usar PM2 — restart automático, log rotation: 
    pm2 start dist/worker.js --name "jobsync-worker" --restart-delay 5000
  2. Docker restart policy — se rodar em container: 
    restart: unless-stopped
  3. ProcessError handler — em startWorkers(): 
    process.on('unhandledRejection', (err) => {
  console.error('Unhandled rejection:', err);
  process.exit(1); // deixa PM2 restartar
});
  4. Health check de workers — endpoint que verifica se workers estão rodando (BullMQ worker.isRunning())
  5. Alert on worker down — Sentry captura process.on('uncaughtException') com notify Slack
  6. Queue metrics — se queue.getWaitingCount() sobe sem workers ativos, alertar

📈 Diagrama de Recuperação

Worker crash
    │
    ├─► PM2 detecta exit code != 0
    │   └─► restart after 5s
    │       └─► worker.close() anteriores? (BullMQ limpa connections)
    │
    └─► Jobs na fila:
        PENDING → são reprocessados (idempotency?)
        ACTIVE → BullMQ reassigna após timeout? (Stalled checker)

🔍 Checklist

  • Workers gerenciados por PM2? ❌
  • Docker restart policy? ❌
  • Graceful shutdown (SIGTERM) tratado? ✅
  • unhandledRejection/exception logado? ⚠️
  • Health check workers? ❌
  • Alertas de worker down? ❌
