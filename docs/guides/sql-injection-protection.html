SQL Injection Protection

Prisma ORM vs raw queries, defesa em profundidade

🛡️ Status Atual

Protegido ✅ — Sistema usa exclusivamente Prisma ORM, que usa parameterized queries por baixo dos panos.

✅
Prisma ORM

Todas queries usam Prisma Client, que gera queries parametrizadas automaticamente.

📊 Exemplos de Queries Seguras

// ✅ SEGURO - Prisma
const job = await prisma.job.findFirst({
  where: {
    id,
    organizationId  // Parâmetros são sanitizados automaticamente
  }
});

// ✅ SEGURO - createMany com array
await prisma.job.createMany({
  data: jobsToCreate,
  skipDuplicates: true
});

// ✅ SEGURO - raw com parâmetros
await prisma.$queryRaw\`SELECT * FROM jobs WHERE id = \${jobId}\`;
//              ↑ Prisma faz interpolação segura

⚠️ Possíveis Riscos

  • $queryRaw interpolado — se usar template literals com variáveis:
// ❌ PERIGOSO - NÃO FAZER ISSO
await prisma.$queryRaw\`SELECT * FROM jobs WHERE id = '\${userInput}'\`;
//                 ↑ userInput pode conter '; DROP TABLE jobs; --

// ✅ SEGURO - usar $queryRaw com parâmetros
await prisma.$queryRaw\`SELECT * FROM jobs WHERE id = \${userInput}\`;
                       // ↑ Prisma treata como parâmetro, não string interpolation
  • Debug logs — não logger o input do usuário diretamente em queries
  • JSON fields — Prisma sanitiza, mas cuidado com JSON.stringify de user input

📋 Checklist de Segurança

  • ✅ Sempre usar Prisma methods (findFirst, create, update, delete)
  • ✅ Evitar $queryRaw — usar apenas se estritamente necessário
  • ✅ Se usar $queryRaw — usar parameterized syntax
  • ✅ Organization ID sempre presente — isolamento multi-tenant
  • ⚠️ Input logging — sanitizar antes de log

✅ Recomendações

  1. ESLint rule — bloquear $queryRaw/template literals perigoso
  2. Code review — qualquer raw query precisa de aprovação
  3. PG parameters logging — ativar para audit de queries
  4. WAF — Web Application Firewall pode bloquear SQLi patterns

🔍 Verificação

// Busca por $queryRaw no código
grep -r "\$queryRaw" src/

Se output incluir ${ (template interpolation), revisar urgente.

🔍 Checklist

  • Usa Prisma exclusively? ✅
  • $queryRaw reviewed? ⚠️ (verificar)
  • Template literals safe? ✅
  • WAF com SQLi rules? ❌
  • Audit logging de queries? ❌
