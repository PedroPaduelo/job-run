Integration Testing Strategy

Testes com banco real, BullMQ, Redis

🧪 O que Testar

  • Job creation flow — POST /api/jobs → DB insert → BullMQ enqueue
  • Worker processing — job na fila → processJob → webhook mock → callback
  • Retry logic — webhook falha 3x → status DEAD
  • Queue pause/resume — afeta workers?
  • Multi-tenancy — org A não vê jobs da org B

🔧 Setup de Testes

// vitest.config.ts
export default defineConfig({
  test: {
    environment: 'node',
    setupFiles: ['./tests/setup.ts'],
  },
});

Usar banco de testes PostgreSQL separado (ou SQLite em memória).

✅ Recomendações

  1. Test real DB — não mock Prisma, usar test container
  2. Test real Redis — ou mock doBullMQ usando bullmq-test-utils
  3. Mock webhooks — usar nock para interceptar fetch
  4. Fixtures — criar factories para Organization, Queue, Job
  5. Coverage — mirar >80% em services

🔍 Checklist

  • Testes unitários existentes? ⚠️
  • Testes de integração? ❌
  • Mock de外部API? ❌
  • Test DB configurado? ❌
  • CI roda testes? ❌
