Sistema de Permissões (RBAC)

Controle de acesso baseado em papéis e permissões granulares

Visão Geral

O JobSync usa RBAC: cada usuário tem uma role dentro de uma organização (OWNER, ADMIN, MEMBER, VIEWER). As roles mapeiam para conjuntos de permissões.

Permissões são strings no formato recurso:ação (ex: jobs:create). O backend verifica via hasPermission(userPermissions, required).

Roles e Permissões Padrão

RolePermissões
OWNERTodas (*)
ADMINJobs: create/read/update/delete/retry/cancel; Queues: create/read/update/delete/pause; ApiKeys: create/read/delete
MEMBERJobs: create/read/retry/cancel; Queues: read
VIEWERJobs: read; Queues: read

Veja o mapeamento completo em src/lib/auth.ts na constante ROLE_PERMISSIONS.

Tipos de Permissões

  • jobs:create
  • jobs:read
  • jobs:update
  • jobs:delete
  • jobs:retry
  • jobs:cancel
  • queues:create
  • queues:read
  • queues:update
  • queues:delete
  • queues:pause
  • apikeys:create
  • apikeys:read
  • apikeys:delete
  • admin:* (wildcard)
  • org:settings

Verificação

Função hasPermission(permissions, required):

if (userPermissions.includes('*')) return true;
if (userPermissions.includes('resource:*')) return true;
return userPermissions.includes(required);

Rotas usam o middleware requirePermission('jobs:create') que chama hasPermission e retorna 403 se negado.

Contexto de Permissões Frontend

O hook usePermissions() (da SDK) permite mostrar/esconder botões condicionalmente. Exemplo:

const { hasPermission } = usePermissions();
if (hasPermission('jobs:create')) {
  // Botão Novo Job aparecerá
}
