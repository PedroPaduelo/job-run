JobSync API Reference

Documentação completa de todos os endpoints REST da API JobSync com exemplos de requisição e resposta.

Autenticação e Autorização

Todos os endpoints da API (exceto /health e /ready) requerem autenticação. O JobSync suporta dois métodos de autenticação:

Headers Obrigatórios

Header Descrição Exemplo
Authorization Token de autenticação (JWT ou API Key) Bearer eyJhbGciOiJIUzI1NiIs... ou Bearer jsk_abc123...
X-Organization-Id ID da organização (opcional, pode estar no token) org_123abc456
ℹ️
Métodos de autenticação suportados:

JWT: Tokens OAuth 2.0 com PKCE do Auth Service.
API Key: Chaves geradas via POST /api/apikeys com prefixo jsk_.

Respostas de Erro de Autenticação

CódigoMensagemCausa
401Missing authorization headerHeader Authorization ausente
401Invalid tokenToken JWT inválido ou expirado
401API key is inactiveAPI Key foi revogada
401API key has expiredAPI Key expirou (expiresAt)
401IP not allowedIP não está na allowlist da API Key
403Forbidden: Missing permission: ...Usuário sem permissão necessária

Jobs

Endpoints para gerenciar jobs na fila de processamento.

POST /api/jobs — Criar Job Único

POST /api/jobs Cria um novo job na fila

Autenticação: JWT ou API Key (permissão: jobs:create)

Request Body (Zod Schema)

{
  "queue": "email-notifications",           // string, obrigatório
  "webhookUrl": "https://api.example.com/", // string URL, obrigatório
  "webhookMethod": "POST",                   // GET|POST|PUT|PATCH|DELETE, default: POST
  "callbackUrl": "https://callback.example.com/", // string URL, opcional
  "payload": { "to": "user@example.com" },  // object, obrigatório
  "userId": "user_123",                      // string, opcional
  "externalId": "ext_456",                   // string, opcional
  "priority": 10,                            // 0-100, default: 0
  "scheduledAt": "2024-01-15T10:00:00Z",   // ISO datetime, opcional
  "metadata": { "source": "api" }           // object, opcional
}

Response (201 Created)

{
  "id": "job_abc123",
  "status": "PENDING",
  "queue": "email-notifications",
  "createdAt": "2024-01-15T09:30:00.000Z"
}

Erros

CódigoMensagem
400Validation error (schema Zod)
400Queue is not active

Efeitos Colaterais

  • Cria registro na tabela jobs
  • Adiciona job na fila BullMQ
  • Cria fila automaticamente se não existir
  • Log de auditoria: job.create

Exemplo cURL

curl -X POST https://api.jobsync.com/api/jobs \
  -H "Authorization: Bearer jsk_abc123..." \
  -H "Content-Type: application/json" \
  -H "X-Organization-Id: org_123" \
  -d '{
    "queue": "email-notifications",
    "webhookUrl": "https://api.example.com/send",
    "webhookMethod": "POST",
    "payload": { "to": "user@example.com", "subject": "Hello" }
  }'

POST /api/jobs/bulk — Criar Múltiplos Jobs

POST /api/jobs/bulk Cria múltiplos jobs de uma vez

Autenticação: JWT ou API Key (permissão: jobs:create)

Request Body (Zod Schema)

{
  "queue": "batch-processing",
  "jobs": [
    {
      "webhookUrl": "https://api.example.com/1",
      "webhookMethod": "POST",
      "payload": { "id": 1 },
      "priority": 5,
      "metadata": { "batch": "A" }
    },
    {
      "webhookUrl": "https://api.example.com/2",
      "webhookMethod": "POST",
      "payload": { "id": 2 },
      "priority": 5
    }
    // ... até 1000 jobs por requisição
  ]
}

Response (201 Created)

{
  "created": 100,
  "failed": 0,
  "queue": "batch-processing"
}

Erros

CódigoMensagem
400Validation error
400Queue is not active

Efeitos Colaterais

  • Cria registros em massa na tabela jobs
  • Adiciona jobs em lote na fila BullMQ
  • Log de auditoria: job.bulk_create

Exemplo cURL

curl -X POST https://api.jobsync.com/api/jobs/bulk \
  -H "Authorization: Bearer jsk_abc123..." \
  -H "Content-Type: application/json" \
  -H "X-Organization-Id: org_123" \
  -d '{
    "queue": "batch-processing",
    "jobs": [
      { "webhookUrl": "https://api.example.com/1", "payload": { "id": 1 } },
      { "webhookUrl": "https://api.example.com/2", "payload": { "id": 2 } }
    ]
  }'

{

Rate Limiting

O JobSync aplica rate limiting a todas as requisições autenticadas:

  • Padrão: 1000 requisições por minuto por organização
  • API Keys: Podem ter rate limit customizado (campo rateLimit)

O rate limit é baseado no X-Organization-Id header ou IP da requisição.

HeaderDescrição
X-RateLimit-LimitLimite de requisições
X-RateLimit-RemainingRequisições restantes
